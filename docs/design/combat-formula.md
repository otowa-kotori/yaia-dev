# 属性体系与战斗公式

> **状态**：这是现行设计目标，不等于当前 runtime 已完全实现。
>
> 当前 ATB、`PATK / MATK / PDEF / MRES / SPD` 体系已经落地；物理伤害函数本身仍在从旧 `phys_damage_v1` 迁移到新的“破甲线”方案。

---

## 1. 属性体系

### 一级属性

| 属性 | 主要影响 |
|------|----------|
| **STR** | 战士向物理成长、部分采集效率 |
| **DEX** | 巡林客向物理成长、部分采集效率 |
| **INT** | 法术伤害、治疗量、牧师成长 |

> `WIS` 已砍掉，牧师共用 `INT`。

### 导出属性

| 属性 | 说明 |
|------|------|
| **PATK** | 物理攻击力，核心由武器物攻与主属性缩放得到 |
| **MATK** | 魔法攻击力，核心由武器魔攻与 `INT` 缩放得到 |
| **PDEF** | 物理防御；纯装备护甲值，不由属性缩放 |
| **MRES** | 魔法抗性；百分比减伤 |
| **SPD** | 独立速度属性，影响 ATB 充能速率 |
| **HP / MP** | 生存与资源池 |

### 护甲分层

- **重甲**：战士
- **中甲**：巡林客、牧师
- **轻甲**：法师

`PDEF` 的设计原则是：**主要来自装备，不来自主属性成长**。

---

## 2. 面板攻击的设计原则

### 物理

目标语义仍然是：

```text
PATK ≈ 武器物攻 × (1 + 主属性缩放)
```

当前实现为了控制膨胀，内容层实际上做成了：

- 一级属性先汇聚到 `PHYS_POTENCY`
- `PATK` 再由 `WEAPON_ATK × f(PHYS_POTENCY)` 派生

这属于**实现细节**；对设计来说，重要的是下面两件事：

1. 武器必须始终有存在感
2. 主属性要放大武器，而不是把武器彻底淹没

### 魔法

```text
MATK ≈ 武器魔攻 × (1 + INT 缩放)
```

同理，当前实现也通过 `MAG_POTENCY` 做了一层中间聚合，但设计目标不变：

- 武器是底座
- `INT` 放大魔法输出

---

## 3. 速度（SPD）

`SPD` 是独立速度属性，不再写成“升级 + DEX + 装备”的派生值。

它的职责很单纯：

- 决定 ATB 能量获取速率
- 决定开场先手优势
- 不直接决定物理伤害或魔法伤害

当前 runtime 已经使用独立 `SPD` 驱动 ATB。

---

## 4. 物理伤害：目标体验已经变了

旧文档里的 `return-to-line` 公式表达的是“允许低攻打高甲时打成 0，并在破甲后逐渐回线”。这不再是当前目标设计。

### 现行目标

物理伤害要给出明确的**破甲线体验**：

- 打不过护甲时，手感要明显难受
- 但不应该长期落到完全 0 伤害
- 一旦破甲，成长反馈要明显改善
- 技能系数始终是独立乘数

### 当前候选表达

更接近现行目标的写法是：

```text
BaseDmg = PATK × skillMul
EffDef  = max(0, PDEF - PEN)
Damage  = max(BaseDmg × floorPct, BaseDmg × 100 / (100 + EffDef))
```

其中：

- `floorPct` 目前倾向约 10%
- `PEN` 可以来自特定技能 / debuff，而不是做成默认全局属性

这个写法的语义是：

- **没破甲**：伤害显著吃瘪，但不完全归零
- **破甲后**：伤害开始线性变舒服
- **技能系数**：仍然通过 `skillMul` 单独控制，不跟护甲规则耦死

### 为什么这样改

因为我们真正想要的不是“永远能慢慢磨死任何怪”，而是：

- 有明显的区域数值门槛
- 物理职业确实会被高甲怪卡住
- 但挂机反馈不能差到只剩一连串 0

---

## 5. 多段物理与巡林客问题

多段物理的核心约束没有变：

- 总 DPS 可以很高
- 但每段都要单独过护甲
- 所以高甲目标会把多段技能打废

这正是巡林客职业区分的来源：

- 清软目标：多段强
- 打硬目标：需要减防、DoT、单段强化或团队配合

也就是说，**巡林客的弱点不是数值不够，而是伤害管线天然怕高甲**。

---

## 6. 魔法伤害

魔法仍然走百分比减伤路线：

```text
Damage = MATK × skillMul × (1 - MRES)
```

设计语义：

- `MRES` 是百分比减伤
- 魔法天然不会出现“完全打不动”的体验
- 降低 `MRES` 的手段应该比物理减防稀少得多
- 魔法职业对高甲目标天然更稳定

这保证了物理 / 魔法两条伤害管线不是同构的。

---

## 7. 物理 vs 魔法的分工

| | 物理 | 魔法 |
|---|---|---|
| 防御形式 | `PDEF`，强调破甲线 | `MRES`，百分比减伤 |
| 体验 | 会被高甲卡住 | 更稳定，不易完全失效 |
| buff / debuff 收益 | 可能出现质变 | 更偏量变 |
| 多段交互 | 高甲时会被强烈克制 | 不会因为“分段”天然吃双重亏 |

---

## 8. 与当前实现的同步情况

### 已同步

- `STR / DEX / INT` 三主属性
- `PATK / MATK / PDEF / MRES / SPD` 五个核心导出属性
- `WIS` 已删除
- `SPD` 已经是独立速度属性
- ATB 已经由 `SPD` 驱动

### 尚未同步

- runtime 里的 `phys_damage_v1` 仍是旧实现，不是本文的目标公式
- 内容层还没有系统化的 `PEN` / 破甲技能族
- 巡林客 / 牧师职业内容还未真正接入，无法完整验证这套公式在实战中的职业分工

---

## 9. 结论

当前这份文档的职责不再是给出一条已经定死的旧公式，而是明确三件事：

1. **属性体系已经换了**：`ATK/DEF/WIS` 时代结束了
2. **速度语义已经换了**：`SPD` 是独立的 ATB 速度属性
3. **物理伤害目标已经换了**：要的是“破甲线 + 低保底”，不是旧的 `return-to-line` 原样照搬
