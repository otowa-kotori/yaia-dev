## 1. 属性体系

### 一级属性

| 属性 | 影响 |
|------|------|
| **STR** | 物理伤害（战士）、采矿效率 |
| **DEX** | 物理伤害（巡林客）、速度、伐木效率 |
| **INT** | 魔法伤害、治疗量、采药效率 |

> WIS 砍掉，牧师共用 INT。

### 导出属性

| 属性 | 来源 | 说明 |
|------|------|------|
| **面板 PATK** | 武器 + 主属性 + 被动 | 面板白字物理攻击力 |
| **面板 MATK** | 武器 + INT + 被动 | 面板白字魔法攻击力 |
| **PDEF** | 纯装备（护甲），不由属性缩放 | 重甲 > 中甲 > 轻甲 |
| **MRES** | 百分比减伤，来源待定 | 有上限（如 80%） |
| **SPD** | 升级 + DEX + 装备 | 影响行动频率 |
| **HP / MP** | 升级 + 装备 | 直接值 |

## 2. 物理伤害公式（已确定方向）

### 面板攻击力

```
面板 PATK = 武器ATK × (1 + k × stat^0.5) × 被动乘数 + flat加成
```

- `k ≈ 0.3`，`stat^0.5` 即 √主属性（sqrt 混合方案）
- 被动乘数：天赋/buff 等提供的百分比加成
- flat 加成：装备副属性等提供的固定值

**核心性质**（实验验证）：
- 换武器跳跃感前后期一致（~43% 提升）
- 膨胀比可控（~8.4 倍 Lv1→Lv25）
- 武器始终占面板 30%+，不会被属性淹没
- sqrt 天然防膨胀——属性 10→40 乘数翻倍，40→90 只再涨 40%

### 破甲系数（ratio-power 方案）

```
有效攻击 = 面板PATK × 技能系数

excess = max(0, PDEF / 有效攻击 - 1)
破甲系数 = max(0.1, K / (K + excess^p))
```

- `K ≈ 0.8`，`p ≈ 1.5`（待联调）
- floor = 0.1（保底 10% 伤害）

**为什么选 ratio-power**：

三段式体验——

| excess 范围 | 伤害比例 | 玩家感受 |
|-------------|---------|---------|
| 0（已破甲）| 100% | 正常伤害 |
| 0~0.3（刚超破甲线）| 85%~95% | "还行，差一点点" ← **缓冲平台** |
| 0.5~1.5（过渡区）| 85%→30% | "明显变弱了，该换装备" ← **核心体验区** |
| >2.0（远超破甲线）| <20%→floor | "真的打不动" |

对比其他方案：
- Hyperbolic（p=1）：没有缓冲平台，破甲线一过就开始衰减；长尾太长
- Exponential：下降太急，过渡区太窄，过于二元
- **Power（p>1）**：p 控制缓冲区大小，K 控制整体宽松度。两个参数，精确调控

**按比例衰减**（非绝对差值）：
- "2 倍防御就是一样难打"，无论装备等级高低
- 实验验证：不同武器 ATK 下，相同 PDEF/ATK 比例的衰减曲线**完全重合**

### 最终伤害

```
最终伤害 = 有效攻击 × 破甲系数 × 暴击乘数 × 其他buff
```

### 多段攻击交互

每段有效攻击 = 面板PATK × 段系数，**独立代入破甲公式**。

多段技能段系数 < 1.0 → 有效攻击下降 → PDEF/有效攻击比例上升 → 破甲系数降低。
高系数技能段系数 > 1.0 → 有效攻击上升 → 更容易破甲。

```
面板 40, PDEF 35:
  重击(×1.5): 有效 60 → 100% → 伤害 60
  连射(2×0.65): 有效 26 → ~70% → 每段 18.2 → 合计 36.4
  → 高甲: 重击 >> 连射

面板 40, PDEF 10:
  重击: 有效 60 → 100% → 60
  连射: 有效 26 → 100% → 两段 52
  → 低防: 差距缩小 (60 vs 52, 仅多 15%)
```

**不需要额外机制**——公式数学性质天然实现"多段打软目标强、碰高甲乏力"。

### 换武器跳跃感

武器 ATK 是面板中的乘法项 → 换武器**同时**提升面板和破甲能力。

```
Lv15 战士, 铜剑(ATK 8) → 面板 22.7 → vs PDEF=35: 破甲~60% → 伤害 13.6
Lv15 战士, 铁剑(ATK 14) → 面板 39.8 → vs PDEF=35: 破甲 100% → 伤害 39.8
→ 换武器: 13.6 → 39.8 (接近 3 倍跳跃，含破甲质变)
```

## 3. 魔法伤害公式

```
面板 MATK = 武器MATK × (1 + k × INT^0.5) × 被动乘数 + flat加成
最终伤害 = 面板MATK × 技能系数 × (1 - MRES%) × 暴击 × buff
```

- MRES% 是百分比减伤，有上限（如 80%）
- 天然无零伤害问题
- 降低 MRES 的技能非常少 → 魔法伤害相对稳定（量变非质变）

### 物理 vs 魔法的核心差异

| | 物理 | 魔法 |
|---|---|---|
| 防御形式 | PDEF（绝对值，ratio-power 衰减）| MRES%（百分比减伤）|
| 破甲/破抗 | 质变（从打不动到打得动）| 量变（从少到多）|
| buff/debuff 影响 | 大（改变破甲状态）| 小（改变减伤百分比）|
| 多段惩罚 | 有（段系数降低有效攻击）| 无（MRES% 不受段系数影响）|
| 高防怪 | 物理噩梦 | 魔法天然克制 |
| 高魔抗怪 | 物理天然克制 | 魔法伤害降低但不归零 |

## 4. PDEF 来源

PDEF 纯装备来源，不由属性缩放。

| 护甲类型 | 可穿戴 | PDEF 系数 |
|----------|--------|-----------|
| 重甲 | 战士 | 最高 |
| 中甲 | 巡林客、牧师 | 中等 |
| 轻甲 | 全职业 | 最低（可能有 INT/MP 加成）|

升级不提升 PDEF → 挨打疼了只能换更好的护甲或强化护甲。

## 附录：探索过程

### A. 攻击力公式探索

> 脚本：`scripts/atk_formula_explore.py`  
> 图表：`docs/design/plots/atk_formula_explore.png`

测试了 5 种公式：纯乘法、加法、sqrt 混合、log 混合、幂函数。

淘汰理由：
- 纯乘法：膨胀 13 倍，武器后期占比仅 25%
- 加法：后期换武器只有 14% 提升，无感
- 幂函数：stat=0 时退化

sqrt 和 log 性质接近，选 sqrt 因为新手阶段感知差异更好。

### B. 破甲线探索

#### B1. 绝对差值方案（已放弃）

测试了 4 种衰减函数（线性/反比例/指数/Sigmoid）基于 `|武器ATK - PDEF|`。

问题：同样"2 倍防御"，高等级时绝对差值更大 → 衰减更严重。不同装备等级下体验不一致。

#### B2. 比例方案（已采纳）

> 脚本：`scripts/armor_ratio_explore.py`  
> 图表：`docs/design/plots/armor_ratio_explore.png`

衰减基于 `PDEF / 有效攻击` 比例。缩放一致性实验验证通过。

测试了 ratio-hyperbolic、ratio-exp、ratio-power 三种变体。

选择 ratio-power 的理由：
- Hyperbolic(p=1)：无缓冲平台，长尾太长
- Exponential：过渡区太窄，过于二元
- **Power(p>1)**：缓冲平台 + 急转直下 + floor 的三段式体验最佳

#### B3. 破甲判定值的选择

最初方案：破甲只看武器 ATK → 多段攻击不被惩罚 → 不符合设计意图。

改为：破甲看**面板 PATK × 技能系数**（有效攻击）→ 多段天然被惩罚，且玩家直觉友好。
