## 1. 属性体系

### 一级属性

| 属性 | 影响 |
|------|------|
| **STR** | 物理伤害（战士）、采矿效率 |
| **DEX** | 物理伤害（巡林客）、速度、伐木效率 |
| **INT** | 魔法伤害、治疗量、采药效率 |

> WIS 砍掉，牧师共用 INT。

### 导出属性

| 属性 | 来源 | 说明 |
|------|------|------|
| **面板 PATK** | 武器 + 主属性 + 被动 | 面板白字物理攻击力 |
| **面板 MATK** | 武器 + INT + 被动 | 面板白字魔法攻击力 |
| **PDEF** | 纯装备（护甲），不由属性缩放 | 重甲 > 中甲 > 轻甲 |
| **MRES** | 百分比减伤，来源待定 | 有上限（如 80%） |
| **SPD** | 升级 + DEX + 装备 | 影响行动频率 |
| **HP / MP** | 升级 + 装备 | 直接值 |

## 2. 物理伤害公式（`return-to-line` 方案）

### 面板攻击力

```
面板 PATK = 武器ATK × (1 + k × stat^0.5) × 被动乘数 + flat加成
```

- `k ≈ 0.3`，`stat^0.5` 即 √主属性（sqrt 混合方案）
- 被动乘数：天赋 / buff 等提供的百分比加成
- flat 加成：装备副属性等提供的固定值

**核心性质**（实验验证）：
- 换武器跳跃感前后期一致（~43% 提升）
- 膨胀比可控（~8.4 倍 Lv1→Lv25）
- 武器始终占面板 30%+，不会被属性淹没
- sqrt 天然防膨胀——属性 10→40 乘数翻倍，40→90 只再涨 40%

### 设计视角：固定怪物，看玩家堆 ATK

物理伤害的讨论改用下面这组变量：

```
有效攻击 = 面板PATK × 技能系数
x = 有效攻击 / PDEF
y = 伤害 / PDEF
```

这样看的是：**同一只怪的 PDEF 固定不变时，玩家随着装备和技能成长，伤害如何增长。**

这个视角比直接看 `PDEF / ATK` 更贴近实际体验，因为玩家感受到的是“我现在这套构筑打这只怪，到底有没有开始动起来”。

### `return-to-line` 公式

若 `PDEF <= 0`，直接视为无甲：

```
伤害 = 有效攻击
```

否则先计算：

```
x = 有效攻击 / PDEF
```

然后使用分段函数：

```
当 x <= 1:
  y = t × x^a

当 x > 1:
  y = (x - 1) + t / (1 + m × (x - 1))
```

其中：

```
a = (1 - t × m) / t
```

约束：

```
0 < t < 1
0 < m < 1 / t
```

最后：

```
伤害 = floor(PDEF × y)
```

### 两个参数的语义

- **`t`**：`x = 1` 时的伤害，也就是 `有效攻击 = PDEF` 时能打出的比例
  - 例如 `t = 0.25` 表示：`ATK = DEF` 时，伤害是 `25% × PDEF = 25% × ATK`
- **`m`**：破甲后回归 `ATK - DEF` 直觉的速度
  - `m` 越小：破甲后的“护甲残余抬升”持续更久，整体更宽松
  - `m` 越大：更快回到 `ATK - DEF`，整体更硬

**第一轮联调建议**：
- `t ≈ 0.25`
- `m ≈ 1.0`

### 为什么选这套

#### 1. 保留 `ATK - DEF` 的长期直觉

当 `x` 很大时：

```
y ≈ x - 1
```

也就是：

```
伤害 ≈ 有效攻击 - PDEF
```

这意味着在装备明显碾压弱怪后，玩家会得到非常直观的反馈：

- 攻击再涨一点，伤害几乎就是线性涨一点
- 弱怪的护甲仍然有数值意义，但不会再主导体验
- “装备好了之后不用担心弱怪伤害”这件事更容易成立

#### 2. 破甲前增长慢，临界区才真正开始动起来

当 `x <= 1` 时，公式是：

```
y = t × x^a
```

这会带来三个性质：

- 低于破甲线时，伤害增长很慢
- 越接近 `x = 1`，增长越明显
- `x = 1` 时不会突然跳到满伤，而是落在设计好的阈值档位 `t`

这比旧的 `ratio-power` 更符合“高甲怪真的卡住物理输出”的目标。

#### 3. 只有一个主弯折，后续更好调

相比“先抬导数、再把导数压回去”的方案，`return-to-line` 的曲线更像一个**单主弯折**：

- 破甲前是一段缓慢上扬的曲线
- 破甲后逐渐接近直线
- 不会出现明显的“第二个拐回来”的视觉折点

而且它只有两个参数：

- `t` 决定“刚打得动时有多痛快”
- `m` 决定“护甲多久失去主导地位”

这比同时调平台高度、过渡宽度、尾部陡峭度更容易掌控。

#### 4. 保留同比例缩放的一致性

公式核心变量是：

```
x = 有效攻击 / PDEF
```

因此它依然保留比例型方案的一个关键优点：

- `ATK = 0.8 × DEF` 在低级区和高级区的体验一致
- `ATK = 1.2 × DEF` 在低级区和高级区的体验也一致

不会回到纯绝对差值方案里“同样差 8 点，但高低级意义完全不同”的问题。

### 推荐锚点（以 `t = 0.25, m = 1.0` 为例）

| `x = ATK / PDEF` | `damage / PDEF` | `damage / ATK` | 体验 |
|---|---:|---:|---|
| 0.5 | 0.031 | 6.2% | 真的打不动 |
| 0.8 | 0.128 | 16.0% | 很难受，但开始看到成长 |
| 1.0 | 0.250 | 25.0% | 刚到可挂线 |
| 1.2 | 0.408 | 34.0% | 明显改善，但护甲仍有存在感 |
| 1.5 | 0.667 | 44.4% | 能打，但不算碾压 |
| 2.0 | 1.125 | 56.2% | 比较舒服 |
| 3.0 | 2.083 | 69.4% | 接近线性碾压区 |

这组点表达的是：

- `ATK = DEF` 不是“破甲后满伤”，而只是**进入能打区**
- 真正的“舒服区”在 `1.5x ~ 2.0x`
- 极端碾压时手感逐步接近 `ATK - DEF`

### 不设百分比保底，允许打成 0

这次方向明确调整为：

- **不再设置全局百分比 floor**
- 伤害按整数结算 `floor(...)`
- 因此在低攻打高甲时，**允许最终出现 0 伤害**

理由：

- 百分比 floor 会随着攻击成长一起变大
- 到中后期容易把“这怪现在不该挂”磨没
- 挂机游戏真正需要的是区域门槛，而不是“任何怪都能慢慢磨死”

如果后续觉得完全 0 伤害的反馈太生硬，优先考虑的不是全局 floor，而是：

- 特定技能自带破甲 / 真伤 / DOT
- 某些重击技能具有更高技能系数
- 对应区域通过内容提示告诉玩家“该换装 / 该换技能了”

### 多段攻击交互

每段有效攻击 = 面板PATK × 段系数，**每段独立代入 `x = 有效攻击 / PDEF`**。

因此多段技能会天然吃到两层惩罚：

- 段系数 < 1，使每段 `x` 下降
- 当 `x < 1` 时，伤害走 `x^a`，下降会更明显

这正好保留了我们要的职业差异：

- 巡林客的多段对软目标仍然强
- 打高甲怪时会明显疲软
- 战士的大系数重击更容易先跨过临界线

示意例子（取 `t = 0.25, m = 1.0`）：

```
面板 40, PDEF 35:
  重击(×1.5): 有效 60, x≈1.71 → 伤害≈30
  连射(2×0.65): 每段有效 26, x≈0.74 → 每段≈3.6, 合计≈7.2
  → 高甲: 重击 >> 连射

面板 40, PDEF 10:
  重击(×1.5): 有效 60, x=6.0 → 伤害≈50
  连射(2×0.65): 每段有效 26, x=2.6 → 每段≈17, 合计≈34
  → 低防: 连射仍有竞争力，但不再无脑接近重击
```

### 换武器跳跃感

武器 ATK 是面板中的乘法项 → 换武器会同时带来两种提升：

- 直接抬高面板 PATK
- 通过 `x = 有效攻击 / PDEF` 把角色往更高的伤害区间推

因此换武器依然会有明显跳跃，但这个跳跃不再表现为“刚过线就满伤”，而是：

- 先从“打不动”进入“可挂线”
- 再从“可挂线”进入“舒服区”
- 最后才逐渐进入“线性碾压区”

这比旧公式更适合挂机成长节奏，因为它让装备升级的收益分布得更自然，而不是在破甲线附近一次性吃光。

## 3. 魔法伤害公式

```
面板 MATK = 武器MATK × (1 + k × INT^0.5) × 被动乘数 + flat加成
最终伤害 = 面板MATK × 技能系数 × (1 - MRES%) × 暴击 × buff
```

- MRES% 是百分比减伤，有上限（如 80%）
- 天然无零伤害问题
- 降低 MRES 的技能非常少 → 魔法伤害相对稳定（量变非质变）

### 物理 vs 魔法的核心差异

| | 物理 | 魔法 |
|---|---|---|
| 防御形式 | PDEF（绝对值，`return-to-line`） | MRES%（百分比减伤） |
| 破甲/破抗 | 临界区质变，破甲后仍持续减伤 | 量变（从少到多） |
| buff/debuff 影响 | 大（改变 `ATK / PDEF` 所处区间） | 小（改变减伤百分比） |
| 多段惩罚 | 有（段系数降低有效攻击） | 无（MRES% 不受段系数影响） |
| 高防怪 | 物理噩梦 | 魔法天然克制 |
| 高魔抗怪 | 物理天然克制 | 魔法伤害降低但不归零 |

## 4. PDEF 来源

PDEF 纯装备来源，不由属性缩放。

| 护甲类型 | 可穿戴 | PDEF 系数 |
|----------|--------|-----------|
| 重甲 | 战士 | 最高 |
| 中甲 | 巡林客、牧师 | 中等 |
| 轻甲 | 全职业 | 最低（可能有 INT/MP 加成）|

升级不提升 PDEF → 挨打疼了只能换更好的护甲或强化护甲。

## 附录：探索过程（只保留关键 insight）

### A. 攻击力公式探索

> 脚本：`docs/design/scripts/atk_formula_explore.py`  
> 图表：`docs/design/plots/atk_formula_explore.png`

核心 insight 只有一条：

- 面板攻击需要同时满足“换武器有明显跳跃”和“后期不被属性完全淹没”
- 纯乘法后期膨胀太快，加法后期换武器无感
- 因此最终选 `sqrt` 混合：既保留武器存在感，又控制数值膨胀

### B. 物理伤害探索

> 脚本：`docs/design/scripts/armor_ratio_explore.py`, `docs/design/scripts/phys_damage_explore_v3.py`, `docs/design/scripts/phys_damage_explore_v4.py`  
> 图表：`docs/design/plots/armor_ratio_explore.png`, `docs/design/plots/phys_damage_progression_view.png`, `docs/design/plots/phys_damage_progression_derivative.png`, `docs/design/plots/phys_damage_simple_compare.png`, `docs/design/plots/phys_damage_return_speed_sweep.png`, `docs/design/plots/phys_damage_fixed_pdef_examples.png`

这轮探索最后沉淀出 4 个关键 insight：

- **视角要换**：讨论护甲公式时，横轴应看 `ATK / PDEF`，主图应看 `damage / PDEF`，因为玩家体验的是“同一只怪在我变强后什么时候开始动起来”
- **旧 ratio-power 不合适**：它更像“未破甲惩罚函数”，不是完整护甲函数；一旦过线，护甲作用衰减得太快
- **曲线形态要单主弯折**：阈值附近如果先抬导数、再压回去，视觉上会出现两个明显折点，不利于长期调参
- **最终目标要清晰**：既要保留 `ATK - DEF` 的长期直觉，又要让 `ATK = DEF` 只落在“刚能打”的阈值档，而不是直接满伤

`return-to-line` 就是围绕这 4 条 insight 收敛出来的。
