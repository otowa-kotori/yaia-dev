## 2. 总体设计

### 四职业总览

| 职业 | 当前内容名 | 主属性 | 护甲 | 定位 |
|------|------------|--------|------|------|
| 战士 | 骑士 | STR | 重甲 | 承伤、拉仇恨、单发物理输出 |
| 巡林客 | 游侠 | DEX | 中甲 | 多段物理、减防、针对软目标高效 |
| 法师 | 法师 | INT | 轻甲 | 稳定魔法输出、AOE / 控场 |
| 牧师 | 圣女 | INT | 中甲 | 治疗、团队辅助、神圣伤害 |

### 已确认的职业规则

- 只做**一转**；零转方案已经砍掉
- 主一级属性只有：`STR / DEX / INT`
- `WIS` 已删除；牧师共用 `INT`
- 护甲分：重甲 / 中甲 / 轻甲
- 角色的速度由独立 `SPD` 控制，不是单纯从 `DEX` 推导出来

### Talent 类型

- **active**：主动技能，占行动窗口
- **passive**：常驻效果，不占行动
- **sustain**：长效姿态，开启后长期生效，同互斥组只能开一个

### TP 设计

**目标设计**：

- 每级获得 **1 TP**
- 还会有额外来源（待定）
- TP 仍然应该稀缺，迫使玩家做取舍

**当前实现并未同步**：

- 代码里仍然是 `totalTp = (level - 1) * 3`
- 这是实现欠账，不应再作为设计文档的依据

## 3. 各职业的设计抓手

### 战士（骑士）

- 高 HP、重甲、最稳的新手职业
- 通过高单发伤害和仇恨控制吃“物理破甲线”的红利
- 可以在“更肉”和“更痛”之间选姿态

### 巡林客（游侠）

- 核心不是单发高伤，而是**多段总 DPS 高**
- 问题是：高甲目标会把每一段都压低
- 因此必须配减防 / DoT / 单段特化，才能打穿硬目标

### 法师

- 走 `MATK × (1 - MRES)` 这条稳定伤害管线
- 不吃物理破甲线，但更依赖 MP 资源管理
- 更适合做群体输出与稳定对高甲敌人解题

### 牧师（圣女）

- 不是“只能奶”的职业
- 必须自带一条可靠的神圣伤害线（例如 `Smite`）
- 组队时偏辅助；单刷时靠神圣伤害与续航技能成立

## 4. 当前已落地内容

### 基础攻击

- 物理职业默认攻击：`talent.basic.attack`
- 魔法职业默认攻击：`talent.basic.magic_attack`

### 骑士 6 天赋（已实现）

| Talent | 类型 | 当前行为 |
|--------|------|----------|
| `重击` | active | 单体高系数物理伤害；MP 消耗随等级提升 |
| `坚韧`（id 仍为 `fortitude`） | passive | 提升生命上限与生命回复 |
| `反击` | passive | 受击且实际掉血后概率反击；当前前置是 `战吼 Lv5` |
| `狂怒` | sustain | `+PATK% / -PDEF%`；当前前置是 `重击 Lv5` |
| `守护` | sustain | `+PDEF% / -PATK%`，并带代伤逻辑；当前前置是 `坚韧 Lv5` |
| `战吼` | active | 给自己上 buff，提高 `AGGRO_WEIGHT` 与 flat `PDEF`；当前前置是 `坚韧 Lv5` |

### 当前落地边界

- 骑士有完整的可学习 talent 列表（`availableTalents`）
- 游侠 / 法师 / 圣女已有 `HeroConfig`、基础属性、起始装备、基础攻击
- 但**它们自己的职业 talent 还没进入内容库**

因此，旧文档里那种“24 个技能都已定完且接近可用”的写法已经不成立了。

## 5. 后续职业方向（未落地，只保留高层约束）

### 巡林客

- 一条线做多段 / 清群
- 一条线做减防 / 单段打硬目标
- 必须显式考虑“高甲时每段都被减伤”的问题

### 法师

- 一条线偏纯输出
- 一条线偏控场 / 减速 / 生存
- 默认攻击存在，但强度要明显低于主技能，避免蓝空时仍接近满效率

### 牧师

- 一条线偏单刷自立（神圣打击、续航）
- 一条线偏组队辅助（治疗、buff）
- 练级不应依赖“别人带”；单人循环必须成立

---

## 6. 新手流程约束

- 第一个角色默认是战士（当前内容：骑士）
- 目标是最简单、最稳、最容易理解的上手体验
- 后续角色解锁后，玩家再逐步接触巡林客 / 法师 / 牧师的复杂玩法
