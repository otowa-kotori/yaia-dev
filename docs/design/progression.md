## 1. 升级 XP 公式

采用 Idleon 式公式：**多项式基础 × 软指数**。

```
XP(lvl) = floor( (a + lvl^p + c·lvl) × (base - min(cap, d·lvl/(lvl+e)))^lvl - offset )
```

### 当前参数

| 参数 | 值 | 含义 |
|------|-----|------|
| a | 8 | 常数底 |
| p | 1.8 | 幂指数，控制前中期增长形状 |
| c | 8 | 线性系数，前 10 级的主力 |
| base | 1.25 | 指数底，驱动后期增长的核心 |
| cap | 0.18 | 后期刹车上限（指数底最终降到 base-cap = 1.07）|
| d | 0.22 | 刹车强度 |
| e | 80 | 刹车生效速度（越大越晚生效）|
| offset | 8 | 前几级的隐形折扣 |

### 公式结构解读

**Part A：`a + lvl^p + c·lvl`** — 表面增长

玩家"看得见"的部分，决定前 15 级手感。

- `c·lvl`（线性项）：前 10 级主力。Lv5 ≈ 40，Lv10 ≈ 80
- `lvl^1.8`（幂律项）：Lv15+ 开始主导。Lv15 ≈ 130，Lv25 ≈ 330

前期体感：每级 XP 多几十，增长平稳，不吓人。

**Part B：`(base - 刹车)^lvl`** — 隐形指数

真正驱动后期变陡的引擎。

- base=1.25：每级乘约 1.25 倍。无刹车时 Lv20 乘数 ≈ 87×，Lv30 ≈ 808×
- 刹车项随等级从 0 爬向 cap=0.18，逐渐压低指数底：
  - Lv5：刹车 ≈ 0.013 → 实际底 ≈ 1.237（还很猛）
  - Lv15：刹车 ≈ 0.035 → 实际底 ≈ 1.215（开始减速）
  - Lv30：刹车 ≈ 0.060 → 实际底 ≈ 1.190（明显减速）
  - Lv80+：刹车 → 0.18 → 实际底 → 1.07（为后续内容留空间）

**一句话**：前 10 级靠线性项快速升级，10 级后指数项接管越来越慢，cap 刹车确保 30 级后不会爆炸。

## 2. 怪物 XP 设计

### 核心原则：怪物 XP 平缓增长

参考 Idleon W1 数据：正式怪物 XP 从 10（Frog）到 75（Wode Board），约 7.5 倍。整个 World 内增长平缓，不是大台阶跳跃。

我们的新手阶段全部在"World 1"内，8 种怪物 XP 从 10 到 85，约 8.5 倍。

| 怪物 | 解锁等级 | XP/只 |
|------|---------|-------|
| 教学怪 | 1 | 3 |
| M1 | 2 | 10 |
| M2 | 5 | 18 |
| M3 | 8 | 28 |
| M4 | 12 | 40 |
| M5 | 16 | 55 |
| M6 | 20 | 70 |
| M7 | 24 | 85 |

> 具体怪物名字和战斗属性见 [monsters.md](./monsters.md)。

### 为什么平缓增长

怪物 XP 增长远慢于升级 XP 需求 → **kills-to-level 自然加速增长**。这是挂机游戏"越来越需要挂更久"的数学来源，不需要人为制造。

## 3. 时间线验证

### 理论纯战斗时间（参数 F，最佳怪物，无效率损失）

| 等级 | 纯战斗时间 | 心流目标 | 效率比 |
|------|-----------|---------|--------|
| 10 | 0.8h | ~2h | ~40% |
| 14 | 2.3h | ~8h | ~29% |
| 20 | 9.0h | ~25h | ~36% |
| 25 | 23.4h | ~55h | ~43% |
| 28 | 40.8h | ~75h | ~54% |

**效率比 = 纯战斗时间 / 心流目标**。约 30-55%，说明玩家约一半时间在战斗挂机，另一半在采集/操作/新角色追赶。后期效率比上升是因为玩家挂机更稳定。

### 战斗节奏假设

| 等级段 | 每波时间 | kill/min |
|--------|---------|---------|
| 1-5 | ~15s | ~4 |
| 6-10 | ~17s | ~3.5 |
| 11-15 | ~20s | ~3 |
| 16-30 | ~24s | ~2.5 |

## 5. 追赶机制

不做独立的经验加成系统。新角色编入高等级队伍打高级怪 = 获得高级怪的满额 XP。组队本身就是追赶手段。

## 附录 B：探索过程

### v1 的错误

假设怪物 XP = `5 + 2*lv + 0.15*lv^2`（二次增长），和升级需求同步增长 → kills-to-level 近似线性 → 总时间仅 12h。

**教训**：怪物 XP 增长必须远慢于升级需求增长。参考 Idleon W1 实际数据修正。

### 参数搜索

测试了 A~H 共 8 组参数。关键发现：
- base < 1.22 的参数组总时间远小于心流目标
- base = 1.28 时 Lv28 纯战斗时间 ≈ 73h，最接近 75h 目标
- base = 1.30 时 Lv28 ≈ 98h，过陡

最终选定 base=1.25 作为起点。与 Idleon Class 线（base=1.208）对比：Lv30 单级 XP 约为 Idleon 的 1.6 倍，考虑到我们将来也会有多角色（和 Idleon 10 角色类似），这个比例合理。
